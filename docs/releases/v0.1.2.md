# AssayAtlas v0.1.2

Released on April 2, 2026.

## Highlights

- Extracted runtime payload normalization into a dedicated validation module.
- Hardened all persisted write paths against blank required strings and malformed structured payloads.
- Removed lingering SQLite resource warnings from the test harness by explicitly closing direct test connections and clients.

## Runtime and API hardening

- `app/runtime_validation.py` is now the shared write-validation seam for:
  - project create/update payloads,
  - dataset updates,
  - figure updates,
  - manuscript updates,
  - and export-job create/update payloads.
- `app/__init__.py` now uses the same allowed-field definitions as the runtime layer, keeping the Flask API contract aligned with the persistence layer.
- `app/runtime.py` now focuses on persistence, event logging, and seeding, while delegating payload checks to the validator module.

## Fixed

- Blank export-job payloads like whitespace-only `title`, `detail`, or `path` now return `400` instead of creating unusable queue items.
- Whitespace-only updates to required text fields such as project names, dataset descriptions, figure summaries, and manuscript narratives are now rejected consistently.
- Structured write payloads such as project team records, milestone lists, figure key metrics, and manuscript sections keep returning deterministic validation errors when malformed.
