# Publication Workflow

## Goal

Make the path from finished analysis to submitted manuscript feel obvious.

Each use-case page now ships with a manuscript bundle that includes:

- SVG and PDF vector figures for drafting.
- High-resolution PNG and TIFF exports for submission portals.
- A figure caption draft.
- A methods blurb.
- A results blurb.
- A submission checklist.
- Source-data CSV files.

## Recommended drafting flow

1. Open the relevant use-case page in the app.
2. Click `Download Manuscript Bundle`.
3. Insert the SVG or PDF figure into the manuscript draft.
4. Start from the included caption draft instead of writing the legend from scratch.
5. Use the methods blurb as the base description of the analysis.
6. Use the results blurb as the first draft of the figure callout in the Results section.
7. Keep the included source-data files next to the draft for supplement assembly and review.

## Which format to use

- `SVG`
  - Best for Word, PowerPoint, Figma, and Illustrator when you want editable vector text and lines.
- `PDF`
  - Best for LaTeX, Overleaf, and publisher workflows that prefer vector PDF figures.
- `PNG`
  - Best for internal drafts, slide decks, and systems that need a high-resolution raster preview.
- `TIFF`
  - Best for journal portals that explicitly request TIFF uploads.

## UX principle

Users should not need to think about export strategy while they are finishing the analysis.

The app should make the default next step obvious:

- draft the paper,
- prepare the supplement,
- or upload the figure.
