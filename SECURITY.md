# Security Policy

## Supported versions

The latest tagged release on `main` is the supported version for security fixes.

## Reporting a vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Instead, report them privately to the repository owner with:

- a short description of the issue,
- reproduction steps or a proof of concept,
- the affected route, file, or workflow,
- and the impact you believe it has.

We will acknowledge receipt, investigate, and coordinate disclosure once a fix is available.

## Security priorities

AssayAtlas treats these as high priority:

- unauthorized file access or data disclosure,
- unsafe export or download behavior,
- dependency vulnerabilities affecting the web app or generator,
- and integrity issues that could silently corrupt figures, summaries, or tutorial outputs.
